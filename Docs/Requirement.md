# 角色
你是一名熟悉 flutter 的開發者

# 任務
我現在想開發一個跨 Mac (Only M-Chips) 與 Windows 的系統列常駐應用程式，目的是為了讓使用者透過快捷鍵呼叫並串接 AI 模型，以達到語音辨識與修飾內容，並自動輸入到游標位置的目的。

# UI/UX 需求
使用者安裝完應用程式，開啟後會要求必要的系統權限（麥克風、輔助使用/無障礙權限以模擬鍵盤貼上）。權限允許後進入主畫面，主畫面左側欄會有如下選項與功能：

- 模型設定
    - 選擇之後，右側會呈現兩個可以收合的 Section，一個是「語音辨識」，另外一個則是「AI 精修」。
    - 針對語音辨識必須加上 red mark 表示為必填。
    - 點開來之後會提供一個 provider list 以供選擇（包含對應的 API Key 輸入框，API Key 需安全儲存）。
    - 語音辨識目前提供的選項有 OpenAI (例如呼叫 Transcribe 模型)。
    - AI 精修目前提供的選項有 Gemini。
    - 各自點了之後又會有模型列表。我希望針對這些設定檔，可以被抽成一份共用的 txt（或 JSON）當中，讓我未來可以針對這些選項去手動新增或修改。

- 字典檔
    - 一個輸入框，輸入完之後右邊有個加入的按鈕，或是可以按下 Enter 直接加入。
    - 輸入框底下會有一個列表，列表內會出現歷史加入過字典檔的字詞，依照 alphabet order 排序。
    - 實際上的編輯就是直接修改 txt 檔案。
    - **用途說明**：字典檔的內容會在發送 API 時，作為提示詞的一部分（提供給語音辨識 Prompt 或 AI 精修 System Prompt），以提高專有名詞的準確度。

- 提示詞修改
    - 針對語音辨識，會有一個系統提示詞區塊，裡面會放目前預設，你先寫 "這是預設提示詞就好"。
    - 針對 AI 精修，會有一個系統提示詞區塊，裡面會放目前預設，你先寫 "這是預設提示詞就好"。

- 設定
    - 開機啟動 (Launch on startup)
    - 設定啟動快捷鍵 (Global Hotkey)
    - 應用程式行為：關閉主視窗時縮小到系統列 (System Tray/Menu Bar) 常駐。

## 操作流程

1. **觸發錄音**：當使用者在任何應用程式中按下設定的快捷鍵之後，會在螢幕的中間下方浮現一個麥克風小圖示與狀態視窗，並開始存取麥克風錄音。
2. **錄音中**：一旦使用者開始講話，圖示內會出現聲音波形；若要取消錄音，可按下 `Esc` 鍵中斷。
3. **結束錄音**：當使用者說完話，再次按下相同的快捷鍵結束錄音。
4. **語音辨識 (Loading 狀態)**：
   - 浮動視窗轉為「處理中」或「辨識中」的動畫提示。
   - 將錄音檔（暫存於系統暫存資料夾）送往語音辨識所選擇的 Provider API 去進行辨識。
   - 辨識完成後取得文字結果，並自動刪除本機暫存音檔。
5. **AI 精修 (可選)**：
   - 系統會檢查使用者有沒有設定 AI 精修。
   - 如果有設定，則呼叫選定的 AI 精修 API，把語音稿當作 User Prompt 傳入；System Prompt 則組合「提示詞修改」中的預設提示詞與「字典檔」內容。
   - 浮動視窗轉為「AI 精修中」的提示狀態。
6. **自動輸出**：
   - 無論是否有經過 AI 精修，當最終文字結果準備好後，自動將文字複製到系統剪貼簿。
   - 系統偵測使用者目前的游標所在位置，模擬鍵盤動作（如 `Cmd+V` 或 `Ctrl+V`）把文字全部貼上。
   - 貼上完成後，自動隱藏/關閉螢幕中間下方的浮動圖示。
7. **例外處理**：
   - 若過程中有 API 錯誤（如連線失敗、API Key 無效），浮動視窗需顯示紅色錯誤提示（例如 "辨識失敗，請檢查網路或 API Key"）約 3 秒後再消失。
